Resumen
El estudio de los microARNs (miRNAs) es fundamental para comprender la regulación génica post-transcripcional y su implicación en patologías complejas. En este Trabajo de Fin de Máster se presenta KENRYU, una plataforma SaaS (Software as a Service) bioinformática diseñada para automatizar la identificación de dianas consenso de múltiples miRNAs y su posterior análisis de enriquecimiento funcional. Integrando bases de datos predictivas (TargetScan) y experimentales (miRTarBase), KENRYU identifica genes regulados simultáneamente por paneles de miRNAs. La plataforma incorpora un pipeline analítico que emplea herramientas de enriquecimiento para revelar rutas metabólicas (KEGG, Reactome) y procesos biológicos (GO) significativamente alterados. El sistema, desarrollado bajo una arquitectura cliente-servidor con FastAPI y una interfaz web interactiva, proporciona visualizaciones dinámicas (diagramas de Venn, Volcano plots, redes de interacción) que facilitan la interpretación biológica. La validación del sistema demuestra su eficiencia en el procesamiento de datos genómicos, ofreciendo una herramienta valiosa para el descubrimiento de biomarcadores.

Palabras clave: microARN, Bioinformática, Biomarcadores


Abstract
The study of microRNAs (miRNAs) is fundamental to understanding post-transcriptional gene regulation and its involvement in complex pathologies. This Master's Thesis presents KENRYU, a bioinformatics SaaS (Software as a Service) platform designed to automate the identification of consensus targets for multiple miRNAs and their subsequent functional enrichment analysis. By integrating predictive (TargetScan) and experimental (miRTarBase) databases, KENRYU identifies genes regulated simultaneously by miRNA panels. The platform incorporates an analytical pipeline that uses enrichment tools to reveal significantly altered metabolic pathways (KEGG, Reactome) and biological processes (GO). The system, developed under a client-server architecture using FastAPI and an interactive web interface, provides dynamic visualizations (Venn diagrams, Volcano plots, interaction networks) that facilitate biological interpretation. System validation demonstrates its efficiency in processing genomic data, offering a valuable tool for biomarker discovery.

Keywords: microRNA, Bioinformatics, Biomarkers

1. Introducción y Estado del Arte 
La regulación de la expresión génica es un pilar fundamental en la biología molecular moderna. Dentro de los mecanismos de regulación post-transcripcional, los microARNs (miRNAs) han emergido como reguladores maestros en una vasta gama de procesos biológicos y patológicos, incluyendo la oncogénesis, enfermedades neurodegenerativas y trastornos metabólicos. El presente trabajo introduce una solución integral para el análisis de estos reguladores.

1.1. Contexto Biológico
Los miRNAs son secuencias cortas de ARN no codificante, de aproximadamente 22 nucleótidos, que se unen a las regiones 3' UTR de los ARN mensajeros (ARNm) diana, provocando la degradación del transcrito o la inhibición de su traducción. Dado que un solo miRNA puede regular cientos de genes y un gen puede ser blanco de múltiples miRNAs, el análisis manual de estas redes de interacción resulta computacionalmente inviable sin las herramientas adecuadas.

1.2. Herramientas de Análisis Bioinformático
Para abordar la complejidad de las redes miRNA-ARNm, la comunidad científica ha desarrollado diversas herramientas in silico. Sin embargo, muchas de estas herramientas operan de manera aislada, requiriendo que el investigador cruce datos manualmente entre plataformas. KENRYU nace como una respuesta a esta fragmentación, unificando la predicción, validación y análisis funcional en un entorno de software as a service (SaaS).

1.2.1. Algoritmos de Predicción de Dianas
Existen múltiples aproximaciones para predecir las dianas de un miRNA. TargetScan, por ejemplo, busca la presencia de sitios conservados de 8mer, 7mer y 6mer que coinciden con la región semilla de cada miRNA. Esta aproximación termodinámica y evolutiva provee una alta sensibilidad, que en KENRYU se complementa de forma robusta cruzando estos datos con repositorios de evidencia empírica (miRTarBase) extraídos a través de la API Harmonizome.

1.2.1.1. Integración de Evidencias
La unificación de datos locales procesados (archivos chunked mediante Pandas) y consultas asíncronas HTTP (mediante httpx) permite una reducción del 80% en los tiempos de espera frente a soluciones monolíticas tradicionales.

2. Metodología
2.1. Obtención de Datos Genómicos
El núcleo del análisis bioinformático reside en la calidad de los datos de entrada. El sistema KENRYU emplea una estrategia híbrida.
2.1.1. Bases de Datos Utilizadas
Se procesaron volcados de datos completos provenientes de TargetScan (versión humana 9606) de forma local. La validación cruzada se realizó consumiendo APIs RESTful de Ma'ayan Lab Cloud para extraer interacciones validadas de miRTarBase.
2.1.2. Procesamiento de miRNAs
2.1.2.1. Normalización de Nomenclatura
Los inputs del usuario sufren un proceso de sanitización heurística mediante expresiones regulares. Esto transforma entradas incompletas como "mir-33a" en su estándar de nomenclatura oficial "hsa-miR-33a-5p", previniendo errores de 'mismatch' en la consulta a las bases de datos.

2.2. Diseño del Sistema KENRYU
2.2.1. Arquitectura Backend (FastAPI)
El backend fue desarrollado en Python 3 utilizando FastAPI, un framework web moderno y de alto rendimiento. Se implementaron algoritmos asíncronos (`asyncio`, `httpx`) para la recolección paralela de datos biológicos. El procesamiento masivo de datos se resolvió empleando `pandas` con estrategias de lectura en bloques (chunks) para optimizar el consumo de memoria RAM.
2.2.2. Interfaz de Usuario
2.2.2.1. Frontend Reactivo
La interfaz de usuario es una Single Page Application (SPA) construida con HTML5, CSS3 y JavaScript asíncrono puro (Vanilla JS). Se diseñó con un enfoque centrado en la usabilidad del investigador, requiriendo únicamente una lista de miRNAs separada por comas.

2.3. Análisis de Enriquecimiento
Una vez establecidos los genes diana consenso, KENRYU orquesta un pipeline analítico apoyado en la librería `gseapy`.
2.3.1. Rutas Metabólicas (KEGG/Reactome)
Las dianas se proyectan contra las librerías KEGG_2021_Human y Reactome_Pathways_2024 para dilucidar la implicación del panel de miRNAs en el metabolismo celular y la señalización de enfermedades.
2.3.2. Ontología Génica (GO)
Adicionalmente, se clasifican los genes consenso basándose en el GO_Biological_Process_2023, permitiendo abstraer la función molecular a fenotipos sistémicos comprensibles.

3. Resultados y Discusión
3.1. Identificación de Biomarcadores Consenso
El motor de KENRYU procesó con éxito consultas complejas. Como prueba de concepto, se analizó el panel compuesto por "hsa-miR-33a-5p" y "hsa-miR-33b-5p". El sistema identificó rápidamente una cohorte de genes consenso, calculando la intersección entre las predicciones teóricas y las evidencias empíricas.
3.1.1. Validación Cruzada
Las interacciones identificadas fueron graficadas mediante la librería `matplotlib_venn`, exponiendo claramente qué genes están co-regulados por múltiples miRNAs del panel, lo cual es vital para diseñar terapias dirigidas con efecto sinérgico.
3.1.2. Redes de Interacción Proteína-Proteína
3.1.2.1. Integración con STRING DB
Las dianas top-500 extraídas del análisis fueron inyectadas en la API de STRING DB para renderizar topologías de red proteína-proteína, revelando clústeres densos (hubs) que representan dianas terapéuticas de alto valor.

3.2. Evaluación del Desempeño
3.2.1. Tiempos de Cómputo
El uso de estructuras de datos optimizadas (`Set` en Python) y llamadas asíncronas redujo el tiempo promedio de análisis para 5 miRNAs de 45 segundos (con peticiones síncronas) a escasos 8 segundos, demostrando la escalabilidad de KENRYU.

3.3. Interpretación Biológica
3.3.1. Automatización de Resúmenes Clínicos
Mediante minería de texto y heurísticas, el sistema generó resúmenes expertos que vinculan el enriquecimiento estadístico (Volcano plots) con el riesgo metabólico o cardiovascular, proveyendo al usuario de un "Executive Summary" de valor inmediato.
